# Зона ответственности
Финальный этап после шагов 1-4: валидация, диагностика, стабилизация.

# Входной контекст
- `/.cursor/plans/timeweb_chat_manager_342cf581.plan.md`
- Все файлы `app/`, `components/`, `lib/`, `types/`, затронутые шагами 1-4

# Что проверить
1. Сквозной поток:
   - UI -> `/api/chat` -> LLM tools -> Timeweb client -> UI.
2. Типизация:
   - отсутствие конфликтов типов между `ai`, `@ai-sdk/react`, `@ai-sdk/openai`, `zod`.
3. UX:
   - корректный стрим ответов;
   - стабильный ввод/отправка;
   - понятный рендер ошибок tools.
4. Надёжность:
   - fallback при ошибках Timeweb API и пустых ответах.

# Ограничения
- Не читать `.env`/секреты.
- Если нужны правки библиотечных интеграций, сверяться через context7.
- Минимизировать рефакторинг: только багфиксы и совместимость.

- Список найденных проблем (severity: high/medium/low).
- Минимальный набор правок для каждого пункта.
- Короткий чеклист ручной проверки перед демо.
- Явно пометь, что проверено, а что осталось предположением.
