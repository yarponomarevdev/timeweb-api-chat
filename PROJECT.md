# Timeweb API Chat — Описание проекта

AI-чат для управления облачными серверами Timeweb Cloud через естественный язык. Вместо веб-панели управления — просто пишешь что нужно сделать.

---

## Технологический стек

| Слой | Технология |
|------|-----------|
| Фреймворк | Next.js 16.1 (App Router) |
| Язык | TypeScript 5 |
| React | React 19 |
| AI SDK | Vercel AI SDK v6 (`ai`, `@ai-sdk/openai`, `@ai-sdk/react`) |
| LLM | OpenAI GPT-4o (настраивается через `OPENAI_MODEL`) |
| Стили | Tailwind CSS v4 |
| UI-компоненты | shadcn/ui + Radix UI |
| Валидация | Zod v4 |
| Markdown | react-markdown + remark-gfm |
| Иконки | Lucide React |

## Возможности

### Управление серверами

| Действие | AI-инструмент | Описание |
|----------|--------------|---------|
| Список серверов | `list_servers` | Все серверы со статусом, IP, ОС, ресурсами |
| Информация о сервере | `get_server` | Детальная информация по ID |
| Создание сервера | `propose_server` → `create_server` | Двухшаговый флоу: сначала подбор конфигурации, затем создание после подтверждения |
| Удаление сервера | `delete_server` | Удаление после явного подтверждения |
| Запуск | `server_action` (start) | Запускает выключенный сервер |
| Выключение | `server_action` (shutdown) | Корректное завершение работы |
| Перезагрузка | `server_action` (reboot) | Перезагрузка работающего сервера |
| Сброс пароля | `server_action` (reset_password) | Сброс root-пароля |
| Переустановка ОС | `server_action` (reinstall) | Полная переустановка операционной системы |

### Информационные запросы

| Действие | AI-инструмент | Описание |
|----------|--------------|---------|
| Тарифы | `list_presets` | Список тарифных планов с ресурсами и ценами |
| Операционные системы | `list_os` | Доступные ОС и их версии |
| Баланс | `get_balance` | Текущий баланс и финансовая информация аккаунта |

## AI-инструменты (tools.ts)

Каждый инструмент — Zod-схема + async-обработчик, который вызывает `lib/timeweb.ts`.

### `list_servers`
Возвращает массив серверов. Рендерится как набор `<ServerCard>` компонентов с кнопками управления.

### `get_server`
Принимает `server_id: number`. Возвращает одну `<ServerCard>`.

### `create_server`
Принимает `os_id: number`, `preset_id: number`, опциональные `name`, `comment`, `avatar_id`. Создаёт сервер и возвращает созданный объект.

### `delete_server`
Принимает `server_id: number`. Удаляет сервер, возвращает подтверждение.

### `server_action`
Принимает `server_id: number`, `action: "start" | "shutdown" | "reboot" | "reset_password" | "reinstall"`. Выполняет действие и возвращает результат.

### `propose_server`
Принимает `os_name?: string`, `preset_id?: number` (или подбирает из доступных). Запрашивает тарифы и ОС, формирует форму `<ServerCreateForm>` с кнопкой создания. **Не создаёт сервер** — только предлагает конфигурацию.

### `list_presets`
Возвращает список тарифных планов в виде таблицы с CPU, RAM, диском и ценой.

### `list_os`
Возвращает список ОС в виде кнопок выбора.

### `get_balance`
Возвращает карточку с балансом, бонусами и статусом оплаты.

## API Route (`app/api/chat/route.ts`)

- **Метод:** POST `/api/chat`
- **Стриминг:** `streamText` из AI SDK, результат через `result.toDataStreamResponse()`
- **Rate limiting:** 20 запросов/минуту на IP (in-memory, сбрасывается при перезапуске)
- **Системный промпт:** инструкции на русском — правила работы с инструментами, флоу создания/удаления, запрет markdown-таблиц (данные рендерятся как UI)
- **Максимум шагов:** `CHAT_MAX_STEPS` (default 8) — защита от бесконечных tool-цепочек

## Клиент Timeweb API (`lib/timeweb.ts`)

Обёртка над Timeweb Cloud REST API v1. Все запросы авторизуются через `Bearer TIMEWEB_TOKEN`.

| Метод | Эндпоинт | Описание |
|-------|---------|---------|
| `listServers()` | GET `/api/v1/servers` | Список серверов |
| `getServer(id)` | GET `/api/v1/servers/:id` | Конкретный сервер |
| `createServer(params)` | POST `/api/v1/servers` | Создание сервера |
| `deleteServer(id)` | DELETE `/api/v1/servers/:id` | Удаление сервера |
| `serverAction(id, action)` | POST `/api/v1/servers/:id/action` | Действие над сервером |
| `listPresets()` | GET `/api/v1/presets/servers` | Тарифные планы |
| `listOS()` | GET `/api/v1/os/servers` | Операционные системы |
| `getBalance()` | GET `/api/v1/account/finances` | Финансы аккаунта |

Хелперы:
- `getServerMainIP(server)` — извлекает основной IPv4 из массива сетевых интерфейсов
- `getStatusLabel(status)` — переводит статус сервера на русский (`on` → "Работает", `off` → "Выключен", etc.)

## UI/UX

### Тёмная тема
Фон `#212121`, акцентный цвет — синий. Полностью тёмная палитра через CSS-переменные Tailwind v4.

### Адаптивность
- **Десктоп:** статичный sidebar слева + чат справа
- **Мобильный:** sidebar как overlay по кнопке-гамбургеру

### Компоненты отображения результатов

**`<ServerCard>`** — карточка сервера:
- Цветной статус-бейдж (зелёный/красный/жёлтый)
- IP-адреса, ОС, CPU/RAM/диск, локация, дата создания
- Кнопки действий в зависимости от статуса:
  - `on`: перезагрузка, выключение
  - `off`: запуск
  - transitional: кнопка обновить статус

**`<ServerCreateForm>`** — форма создания:
- Дропдаун выбора версии ОС
- Карточка с тарифом (CPU, RAM, диск, цена/месяц)
- Кнопка "Создать сервер" — отправляет подтверждение в чат

### Поведение чата
- Сохранение истории в `localStorage` (восстанавливается при перезагрузке)
- Автоскролл вниз только если пользователь уже внизу (scroll-lock)
- Retry для последнего пользовательского сообщения
- Быстрые действия на пустом экране (кнопки в sidebar)
- Стриминг ответов в реальном времени

## Флоу создания сервера

```
Пользователь: "Создай сервер с Ubuntu"
     ↓
AI вызывает propose_server(os_name="Ubuntu")
     ↓
Рендерится <ServerCreateForm> с тарифом и выбором версии ОС
     ↓
Пользователь выбирает версию и нажимает "Создать сервер"
     ↓
В чат отправляется: "Подтверждаю создание: os_id=XXX, preset_id=YYY"
     ↓
AI вызывает create_server(os_id=XXX, preset_id=YYY)
     ↓
Рендерится <ServerCard> с созданным сервером
```

## Флоу удаления сервера

```
Пользователь: "Удали сервер test-server"
     ↓
AI вызывает list_servers() → находит нужный
     ↓
AI просит подтверждение с ID и именем сервера
     ↓
Пользователь подтверждает
     ↓
AI вызывает delete_server(server_id=XXX)
     ↓
Уведомление об успешном удалении
```
